Запустіть проєкт у режимі розробки:

Bash
npm run dev
(Сервер за замовчуванням запускається на порту 3000).

🧪 Як тестувати (Демонстрація різниці)
Для перевірки обох підходів використовуйте будь-який REST-клієнт (Postman, Insomnia, Thunder Client).

Тіло запиту (JSON):

JSON
{
  "bookId": "book-123",
  "userId": "user-456"
}
1. Асинхронний підхід (Рекомендований)
Ендпоінт: POST /api/loans

Результат: Відповідь від сервера надходить миттєво. У консолі можна спостерігати, як NotificationSubscriber перехоплює подію і відправляє лист у фоні (із затримкою), не змушуючи клієнта чекати.

2. Синхронний підхід (Для порівняння)
Ендпоінт: POST /api/loans/sync

Результат: Відповідь від сервера надходить із затримкою ~1.5 секунди. Потік виконання блокується до завершення роботи сервісу нотифікацій.
