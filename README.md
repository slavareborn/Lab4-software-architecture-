# Лабораторна робота 4: Синхронна та асинхронна комунікація між компонентами

## 📖 Опис проєкту
Цей репозиторій містить реалізацію лабораторної роботи №4 з дисципліни "Архітектура програмного забезпечення". Головна мета роботи — практичне освоєння патернів взаємодії між компонентами системи, розділення основної бізнес-логіки та побічних ефектів, а також порівняльний аналіз синхронного та асинхронного підходів.

В якості предметної області використовується **Система управління бібліотекою** (Library Management System).
- **Основна бізнес-дія:** Успішна видача книги читачу (`BorrowBookCommand`).
- **Побічна операція (побічний ефект):** Надсилання email-нотифікації користувачу з деталями позики.

---

## 🛠️ Що було реалізовано

1. **Допоміжний компонент (`NotificationService`):**
   - Виділено в окремий модуль інфраструктурного шару (`infrastructure/notifications`).
   - Має чіткий контракт у вигляді інтерфейсу `INotificationService` в аплікаційному шарі, що задовольняє принцип інверсії залежностей (DIP).
   - Імітує реальну затримку введення-виведення (I/O) тривалістю 1.5 секунди.

2. **Синхронний варіант взаємодії (`BorrowBookHandlerSync`):**
   - Прямий виклик методу сервісу нотифікацій одразу після збереження сутності в базу даних.
   - Операція відбувається в одному потоці виконання.
   - Реалізовано стійкість до збоїв: помилки сервісу нотифікацій перехоплюються через блок `try/catch`. Це запобігає скасуванню або відкоту основної бізнес-операції (книга залишається виданою, навіть якщо email не надіслався).

3. **Асинхронний варіант взаємодії (`BorrowBookHandler`):**
   - Реалізовано внутрішньопроцесну шину подій (`EventBus`) на базі стандартного `EventEmitter` в Node.js.
   - Створено клас події `BookBorrowedEvent`, який є незмінним (`immutable` через `Object.freeze`) та іменований у минулому часі. Подія містить унікальний `eventId` для забезпечення ідемпотентності.
   - Створено підписника (`NotificationSubscriber`), який слухає шину подій та запускає процес нотифікації у фоновому режимі, не блокуючи відповідь користувачу.


## 🚀 Встановлення та запуск

1. **Встановіть залежності:**
   ```bash
   npm install
   ```

2. **Запустіть сервер у режимі розробки:**
   ```bash
   npm run dev
   ```
   За замовчуванням сервер запускається та слухає запити на порту 3000.
